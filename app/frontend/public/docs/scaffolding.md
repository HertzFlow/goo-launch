# example-goo scaffolding

Go + Hardhat + Vite full-stack project: deploy ERC20 contracts via HTTP API and start long-lived goroutines (Agenters) to automatically trigger contract method calls. The frontend provides a user dashboard and a global data view.

## Architecture

Three-service Docker Compose stack:

| Service | Port | Description |
|---------|------|-------------|
| `goo-server` | :8080 | Go backend (Gin) |
| `goo-frontend` | :3000 | User frontend Vite/Nginx SPA |
| `goo-postgres` | :6432 | PostgreSQL 16 |

Target chain: **BSC Testnet**. Blockchain interactions use a temporary deployer key pair pattern (System transfers funds to deployer -> deployer deploys contract -> deployer signs subsequent transactions).

## Core Flow

### Create Agent

```
POST /api/deploy  { "contract": "ExampleToken" }
```

1. Generate a new deployer key pair
2. System address transfers 0.1 tBNB to the deployer
3. Deploy the contract on BSC Testnet using the deployer's private key via Hardhat
4. Start an Agenter goroutine (using the deployer identity to sign subsequent transactions)
5. Persist contract and Agenter information to PostgreSQL

The entire flow has a 120s context timeout; the Hardhat subprocess will be killed on timeout.

### Agenter Auto-Trigger

Each Agenter automatically calls `mintWithMemo` at geometric intervals (10s -> 20s -> 40s -> 80s...) after startup. Memos are generated by the DeepSeek LLM. Agenters are long-lived goroutines managed by `agenter.Manager` (in-memory map + RWMutex). A single user can create multiple Agenters (one per deploy).

### Authentication

Passwordless JWT authentication. `POST /api/auth/login` with `{username}` auto-creates users. A default admin user (username: `admin`) is seeded at startup. Roles: `admin`, `user`. Token expires after 24h.

## Frontend Pages

Navbar links: **Home** | **Dashboard** | **All** | **Logout**

| Page | Path | Description |
|------|------|-------------|
| Home | `/` | Create Agent (deploy contract + start Agenter), Trigger, Status |
| Login | `/login.html` | Passwordless login, just enter a username |
| Personal Dashboard | `/dashboard.html` | Current user's contracts, Agenters, transaction history |
| Global View | `/all.html` | Platform-wide stats + all users/contracts/Agenters/transactions, admin can delete users |

Addresses and Tx Hashes are displayed as clickable links pointing to [BSC Testnet Scan](https://testnet.bscscan.com).

## API Endpoints

### Public

```
POST /api/auth/login              # { "username": "xxx" } → { "token": "..." }
```

### Authenticated (Bearer token)

```
GET  /api/auth/me                 # Current user info
POST /api/deploy                  # Create Agent (deploy contract + start Agenter)
POST /api/agenter/trigger         # Manually trigger Agenter
GET  /api/agenter/status?agenter_id=  # Query Agenter status

GET  /api/my/contracts            # My contracts
GET  /api/my/agenters             # My Agenters
GET  /api/my/transactions         # My transaction history

GET  /api/all/stats               # Global stats (user count/contract count/Agenter count)
GET  /api/all/users               # All users
GET  /api/all/contracts           # All contracts
GET  /api/all/agenters            # All Agenters
GET  /api/all/transactions        # All transaction history
```

### Admin (requires admin role)

```
DELETE /api/admin/users/:id       # Delete user
```

## Quick Start

```bash
# Start all services with a single command
make start-server

# Health check
curl http://localhost:8080/ping

# View logs
make logs

# Stop
make stop-server
```

## Makefile Commands

| Command | Description |
|---------|-------------|
| `make start-server` | Build images and start all services |
| `make stop-server` | Stop all services |
| `make logs` | View server container logs |
| `make clean` | Stop services, remove containers, images, and volumes |
| `make abigen` | Generate Go bindings from contract ABI |

## Environment Variables

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `RPC_URL` | BSC Testnet RPC endpoint |
| `SYSTEM_PRIVATE_KEY` | System wallet private key (used to fund deployers) |
| `CONTRACTS_DIR` | Contract project path inside container (default: `/contracts`) |
| `DEEPSEEK_API_KEY` | DeepSeek LLM API Key (optional, enables memo generation) |
| `JWT_SECRET` | JWT signing secret |
| `NETWORK` | Hardhat deploy target network (default: `bscTestnet`) |

All variables have local development defaults in `go/internal/config/config.go`.

## Database

GORM AutoMigrate automatically creates tables at startup:

| Table | Description |
|-------|-------------|
| `users` | Users (username, role, password hash) |
| `contracts` | Contract deployment records (address, deployer, txHash, network, user_id) |
| `agenter_records` | Agenter state (agenter_id, status, trigger count, last error, user_id) |
| `transaction_logs` | Transaction records (agenter_id, method, memo, txHash, status, user_id) |

## Contracts

- **`ExampleToken.sol`** — ERC20 + Ownable
  - `mint(to, amount)` — Mint tokens
  - `mintWithMemo(to, amount, memo)` — Mint with a memo (emits event)
- Deploy script outputs `{address, txHash}` JSON to stdout, parsed by the Go deployer
